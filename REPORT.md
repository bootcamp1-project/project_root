## 1. 프로젝트 개요

초등학생을 대상으로, 웹 브라우저에서 블록을 조합해 프로그램을 작성하면 TurtleBot3가 순서대로 동작을 수행하는 블록 코딩 기반 로봇 제어 시스템을 5일간 개발했다.

사용자는 스마트폰 또는 PC의 웹 앱에서 블록을 배치하고 실행 버튼을 누르며, 로봇이 명령을 수행하는 동안 현재 실행 중인 블록이 화면에 실시간으로 하이라이트된다. 라이다(LDS-03) 센서를 통해 전진 및 후진 방향의 장애물을 감지하면 로봇이 즉시 안전하게 정지하도록 하여, 교육 환경에서도 안전하게 사용할 수 있도록 설계했다.

## 3. 최종 시스템 아키텍처

```text
① Web Application
   - 블록 편집기
   - 실행 버튼
   - 정지 버튼
   - 실행 상태 표시
        │
        ▼
② Interpreter Node (웹소켓 서버 내장)
   JSON 프로그램을 해석하여 아래 명령을 순차적으로 실행
   - 전진 / 후진
   - 좌회전 / 우회전
   - 부저
   - 대기
        │
        ▼
③ TurtleBot3 및 센서
   - TurtleBot3 (모터 구동부)
   - 라이다(LDS-03) 센서 (전방/후방 20cm 장애물 감지)
   - 부저
```

당초 계획 단계에서는 웹과 ROS 2 사이를 rosbridge_server로 중계할 예정이었으나, 최종적으로는 interpreter_node에 웹소켓 서버를 직접 내장하여 rosbridge 없이 turtlebot3_bringup과 block_robot bringup 두 개의 launch만으로 전체 시스템이 동작하도록 단순화했다.

## 7. 주요 이슈 및 해결

| 이슈 | 원인 | 해결 |
| --- | --- | --- |
| 웹소켓 연결 시 보안 인증서 경고 | 로봇 내부 IP에 대한 `wss` 인증서 미신뢰 | 브라우저에서 로봇 IP로 직접 접속하여 인증서 예외 승인 |
| 회전각 정밀도 부족 (90도 미만 회전) | 배터리 전압 저하 및 환경 상수 불일치 | 배터리 만충 상태에서 반복 측정하여 `TURN_90_SEC = 3.1`로 보정 |
| 회전 중 안전 정지 미작동 | 회전 중에는 거리 임계값을 검사하지 않는 구조 | 회전 중 오검측 리스크를 고려해 '전진/후진 중에만 정지' 정책을 `INTERFACE.md`에 명시 |
| 실행 중 아이의 화면 터치로 프로그램 꼬임 | 실행 중에도 편집 UI가 활성 상태 | 실행 시 UI 잠금 + ROS 단에서도 실행 중 새 프로그램 수신 무시(이중 방어) |
| 장애물 정지 시 영어 로그 노출 | 개발자용 로그를 그대로 사용자에게 노출 | 아동 친화적 안내 모달("앞에 무언가 있어요! 🛑")로 교체 |

## 8. KPT 회고

### Keep (유지할 점)

- 파트별 독립 개발 후 통합하는 방식으로 병렬 개발 속도를 확보한 점
- 실제 아동 사용자 관점을 반영한 UX 테스트(Day 4)를 통해 문제를 조기에 발견한 점
- 웹소켓 서버를 interpreter_node에 직접 내장해 시스템 구조를 단순화한 점

### Problem (아쉬운 점)

- 프로그램 저장/불러오기, 실행 경로 미리보기 등 Should 항목 일부를 완료하지 못한 점
- 회전 중 장애물 감지가 되지 않는 한계가 남아 있는 점
- 초기 웹소켓 인증서 이슈처럼 배포 환경(HTTPS/WSS) 관련 이슈에 대한 사전 검토가 부족했던 점

### Try (다음에 시도할 점)

- 회전 중에도 안전하게 활용할 수 있는 장애물 감지 로직 연구
- 프로그램 저장/불러오기 기능을 다음 개발 주기에 우선 반영
- 인증서/네트워크 이슈를 사전에 점검하는 체크리스트 마련

## 9. 결론 및 기대 효과

이번 프로젝트를 통해 초등학생도 쉽게 사용할 수 있는 블록 코딩 기반 로봇 제어 시스템을 구현했다. 웹과 ROS 2를 직접 연동하는 경량 구조를 설계 및 검증했고, 라이다 센서를 이용한 안전 기능과 실제 사용자 테스트를 통한 UX 개선을 통해 교육 현장에서 안전하게 활용 가능한 형태로 완성했다.

- 초등학생도 쉽게 로봇 프로그래밍을 경험할 수 있는 환경을 제공한다.
- 블록 코딩과 실제 로봇 제어를 연계한 교육 효과를 제공한다.
- ROS 2 기반 시스템 설계 및 구현 경험을 습득할 수 있었다.
- 사용자 중심의 UI/UX 설계와 협업 개발 과정을 경험할 수 있었다.
