# 블록 코딩 로봇 교육 플랫폼

> ROS 2 Humble 기반 TurtleBot3 블록 코딩 교육 플랫폼 프로젝트 기획 문서

---

# 1. 프로젝트 개요

## 프로젝트명

**블록 코딩 로봇 교육 플랫폼**

---

## 개발 목적

본 프로젝트는 초등학생을 대상으로 하는 블록 코딩 교육 환경을 구축하기 위해 기획되었다.

사용자는 웹 브라우저에서 Scratch 방식의 블록을 조립하여 프로그램을 작성하고, 이를 TurtleBot3와 연동하여 실제 로봇을 제어할 수 있다.

기존 교육용 블록 코딩 시스템은 대부분 시뮬레이터 중심으로 구성되어 있어 실제 하드웨어를 직접 제어하는 경험이 제한적이다.

본 프로젝트에서는 ROS 2 기반 통신과 Runtime Interpreter를 이용하여 웹에서 작성한 프로그램을 실제 TurtleBot3가 수행하도록 구현하는 것을 목표로 한다.

---

# 2. 개발 배경

최근 소프트웨어 교육에서는 단순한 코딩 문법 학습보다 문제 해결 능력과 컴퓨팅 사고력을 기르는 교육이 강조되고 있다.

Scratch와 같은 블록 코딩 도구는 초등학생도 쉽게 접근할 수 있는 장점이 있지만, 대부분 화면 속 가상 객체만 제어한다는 한계가 있다.

반면 ROS 2는 실제 로봇을 제어하기 위한 표준 미들웨어이지만 교육용으로 활용하기에는 진입 장벽이 높다.

따라서 본 프로젝트는 Scratch 방식의 사용자 인터페이스와 ROS 2를 결합하여 사용자가 작성한 프로그램이 실제 TurtleBot3에서 실행되는 교육 플랫폼을 구현하고자 한다.

본 프로젝트의 목표는 다음과 같다.

# 6. 시스템 아키텍처

```text
Web Browser

↓

rosbridge_server

↓

ROS2

↓

Interpreter Node

↓

TurtleBot3

↑

LiDAR
```

웹 애플리케이션은 프로그램을 JSON Tree 형태로 생성하여 ROS2에 전달하며, Interpreter Node는 이를 Runtime에 해석하여 TurtleBot3를 제어한다.

---

# 7. 핵심 기능

본 프로젝트의 핵심 기능은 다음과 같다.

## Runtime Tree Interpreter

반복문과 조건문을 Runtime에 직접 해석하는 구조를 적용하였다.

---

## Nested JSON Tree

블록 프로그램을 Tree 구조로 표현하여 중첩 구조를 지원한다.

---

## Runtime Skip & Branch

장애물이 감지되면 현재 이동 블록만 종료하고 다음 실행 흐름으로 이동한다.

---

## LiDAR Safety

LiDAR 데이터를 이용하여 장애물을 감지하고 안전한 주행을 지원한다.

---

## Web Highlight

현재 실행 중인 블록을 실시간으로 웹에서 표시한다.

# 12. 개발 전략

프로젝트는 기능 단위가 아닌 **시스템 통합 중심**으로 개발을 진행하였다.

초기에는 웹, ROS2, 하드웨어를 각각 독립적으로 구현한 후, rosbridge를 이용하여 전체 시스템을 통합하였다.

이후 Runtime Interpreter를 단계적으로 개선하여 반복문과 조건문의 중첩 구조를 지원하도록 확장하였으며, LiDAR 기반 장애물 감지와 Runtime Skip 기능을 추가하여 실제 주행 환경에서도 안정적으로 동작하도록 개선하였다.
