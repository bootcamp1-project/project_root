## 1. 프로젝트 개요

### 1.1 개발 배경

최근 소프트웨어 교육에서는 블록 코딩을 활용하여 프로그래밍 개념을 쉽게 익힐 수 있도록 하는 교육 방식이 널리 사용되고 있다. 하지만 대부분의 블록 코딩은 화면에서만 동작하여 실제 로봇의 움직임을 경험하기 어렵다는 한계가 있다.

본 프로젝트는 ROS 2와 TurtleBot3를 활용하여 초등학생도 쉽게 사용할 수 있는 블록 코딩 기반 로봇 제어 시스템을 개발하는 것을 목표로 한다. 사용자는 웹 브라우저에서 블록을 조합하여 프로그램을 작성하고, 로봇이 해당 명령을 순차적으로 수행하는 과정을 실시간으로 확인할 수 있다.

또한 라이다(LiDAR) 센서를 이용한 장애물 감지 기능을 적용하여 교육 환경에서도 안전하게 사용할 수 있도록 설계한다.

### 1.2 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

- 웹 기반 블록 코딩 인터페이스 개발
- 블록 프로그램을 JSON 형식으로 변환
- Interpreter Node를 이용한 블록 순차 실행
- ROS 2와 웹 간 실시간 통신 구현
- 라이다(LiDAR) 센서를 활용한 안전 기능 구현
- 사용자 중심 UI/UX 설계
- 실제 사용자 테스트를 통한 개선 사항 도출

## 3. 시스템 아키텍처

프로젝트는 크게 다음 네 부분으로 구성된다.

```text
① Web Application
   - 블록 편집기
   - 실행 버튼
   - 정지 버튼
   - 실행 상태 표시
        │
        ▼
② Interpreter Node (웹소켓 서버 내장)
   JSON 프로그램을 해석하여 아래 명령을 순차적으로 실행
   - 전진
   - 좌회전
   - 우회전
   - 부저
   - 대기
        │
        ▼
③ TurtleBot 및 센서
   - TurtleBot3
   - 라이다(LiDAR) 센서
   - 부저
```

당초 계획에서는 웹 앱을 서빙하는 정적 HTTP 서버까지 ROS 2 통합 런치 파일(`bringup.launch.py`) 안에서 함께 띄우는 것을 목표로 했으나, 실제 구현 단계에서는 ROS 2 노드(interpreter_node, ultrasonic_node, buzzer_node)만 launch로 묶고, 웹 서버(`python3 -m http.server 8000`)는 별도 프로세스로 분리해 실행하는 방식으로 확정되었다. 세부 실행 절차는 `README.md`, 변경 배경은 `REPORT.md`(3번, 7번)와 `TEST.md`(3.2)를 참고한다.
