## 📅 Day 2: 파트별 독립 단품 개발

- **일시:** 2026년 7월 7일 (화)
- **회의 목적:** 파트별 UI/코드 스켈레톤 구축 및 개별 동작 검증

### 🌐 웹 파트 (김건호)

- **오늘 한 일:**
  - HTML5와 단순 JS를 이용해 블록을 탭하여 시퀀스 리스트를 생성하는 기본 편집기 UI 레이아웃 구현
  - `roslib.js`를 이용한 로봇 웹소켓(`wss://`) 기본 연결 코드 작성 완료
- **겪은 문제 (Blocker):** GitHub Pages로 배포된 웹 브라우저에서 로봇 내부 IP로의 `wss` 접근 시 보안 인증서 경고로 인해 연결이 차단됨
- **해결 방안:** 실습 가이드 문서 §3~§6에 따라 브라우저에서 로봇 IP 주소로 직접 접속하여 예외 승인(인증서 허용)을 수동으로 진행하여 해결함

### 🤖 ROS 파트 (윤태환)

- **오늘 한 일:**
  - 스타터 코드(`interpreter_node.py`)의 패키지 이식 및 컴파일 검증(`py_compile`) 완료
  - 들어오는 JSON 문자열 데이터를 `json.loads()`로 파싱하여 배열로 적재하는 수신 테스트 완료
- **내일 할 일:** 타이머 주기에 맞춰 각 명령(`forward`, `turn_left` 등)별 실행 시간 소모 및 상태 전환 함수 고도화

- **오늘 한 일:**
  - 라즈베리 파이 기판에 라이다(LDS-03) 센서를 장착하고 10Hz 주기로 `/obstacle_dist` 토픽을 발행하는 독립 노드 구현 완료
  - 부저 모듈 연동 및 테스트 코드로 비프음 발생 확인

## 📅 Day 3: 승부처 - E2E 통합 및 안전장치 결합

- **일시:** 2026년 7월 8일 (수)
- **회의 목적:** 웹-ROS 간 전체 파이프라인 연동 및 예외 상황(장애물 감지) 통합 테스트

### 🔗 전체 통합 현황 (김성수)

- **진척도:** 5블록 프로그램 완주 성공 (M2 달성)
- **연동 항목:** 웹 편집기 실행 ➡️ `/program` JSON 송신 ➡️ 로봇 구동 ➡️ 실시간 블록 전환 상태(`/run_state`) 역전파를 통한 웹 UI 하이라이트 표시 연동 성공

### 🚨 이슈 및 트러블슈팅

**이슈 1: 터틀봇 회전각 정밀도 부족 (90도 미만 회전)** (윤태환)

- *원인:* 배터리 전압 저하 및 환경 상수의 불일치
- *해결:* 터틀봇 배터리를 만충한 상태에서 제자리 회전 시간을 반복 측정하여 `TURN_90_SEC = 3.1` 상수를 미세 보정함

**이슈 2: 회전 동작 중에는 라이다 센서 안전 중단이 작동하지 않음** (김민찬)

- *원인:* 스타터 코드가 `op == 'forward'` 상태에서만 거리 임계값을 검사함
- *해결:* 팀 회의 결과, 회전 중에는 주변 지형지물에 의한 오검측 리스크가 크므로 **'전진 중에만 20cm 기준 중단'** 정책을 유지하기로 `INTERFACE.md`에 명시하고 픽스함

## 📅 Day 4: 반복 블록 확장 및 사용자 테스트

- **일시:** 2026년 7월 9일 (목)
- **회의 목적:** Should 기능 구현 완료 및 외부 인원을 통한 UX 평가

### 💡 Should 기능 (S1 반복 블록) 구현 결과 (김건호)

- **설계 결정:** 로봇 인터프리터 노드의 구조적 과부하를 방지하기 위해, 웹앱(JS) 단에서 반복 루프 구조를 완전히 평탄화(Unrolling)하여 전송함
- **결과:** `[3번 반복: 앞으로, 부저]`의 중첩 블록이 웹에서 `[앞으로, 부저, 앞으로, 부저, 앞으로, 부저]` 형태의 단순 JSON 선형 배열로 변환되어 전달되므로, ROS 노드의 안정성을 유지하면서 기능을 완벽히 수행함

### 🎨 UX 사용자 테스트 실시 (박정호) — M6 달성

- **참가자:** 타 조원 2인 초청 (초등 3학년 역할 수행)
- **도출된 개선점 및 반영:**
  1. **문제:** 로봇이 작동 중일 때 아이가 편집 화면을 임의로 터치하여 프로그램이 꼬임
     - **반영:** 웹앱에서 실행 버튼 클릭 시 UI 전체를 잠금(Lock) 처리하고, ROS 단에서도 실행 중 새 프로그램 수신을 무시하도록 '이중 방어' 구현
  2. **문제:** 장애물 중단 시 웹 화면에 영어 로그가 출력되어 아이가 인지하지 못함
     - **반영:** "앞에 무언가 있어요! 🛑 안전을 위해 로봇이 잠시 쉬고 있어요."라는 아동 친화적 안내 모달 팝업으로 문구 전면 전환
