## 1. 프로젝트 개요

### 1.1 개발 배경

최근 소프트웨어 교육에서는 블록 코딩을 활용하여 프로그래밍 개념을 쉽게 익힐 수 있도록 하는 교육 방식이 널리 사용되고 있다. 하지만 대부분의 블록 코딩은 화면에서만 동작하여 실제 로봇의 움직임을 경험하기 어렵다는 한계가 있다.

본 프로젝트는 ROS 2와 TurtleBot3를 활용하여 초등학생도 쉽게 사용할 수 있는 블록 코딩 기반 로봇 제어 시스템을 개발하는 것을 목표로 한다. 사용자는 웹 브라우저에서 스크래치 방식의 중첩 트리 구조 블록을 조립하여 프로그램을 작성하고, 로봇이 해당 명령을 런타임에 실시간으로 해석하여 수행하는 과정을 시각적으로 교감할 수 있다.

또한, 대각선 사각지대를 완전히 없앤 90도 광폭 부채꼴 라이다(LiDAR) 센서 감지 알고리즘을 적용하여 주행 중 장애물 발견 시 단순 강제 종료가 아닌, 하위 조건 분기 블록으로 부드럽게 제어권을 이양하는 유연한 안전 최적화 플랫폼을 구현한다.

### 1.2 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

- 웹 기반 중첩 드래그 앤 드롭 블록 코딩 인터페이스 개발
- 스크래치 논리와 호환되는 JSON Tree 프로토콜 설계
- ROS 2 Interpreter Node 기반의 런타임 실시간 트리 파싱 엔진 구현
- 실행 중인 하드웨어 연산자(`op`)와 웹 UI 노드 간의 1:1 다이렉트 하이라이트 동기화
- 라이다(LiDAR) 전/후방 90도 부채꼴 스캔을 활용한 사각지대 제로 안전망 전개
- 주행 중 장애물 감지 시 실시간 블록 탈출(Skip) 및 조건 제어권 이양 메커니즘 확립
- 사용자 중심 아동 친화적 UI/UX 설계 및 실제 시뮬레이션 테스트를 통한 개선

## 2. 기능 요구사항

### Must (필수 구현 요구사항)

- 블록 4종 이상 구현 (전진, 후진, 좌회전, 우회전, 대기 등)
- 블록 순차 실행 엔진 구축
- 현재 실행 중인 블록의 1:1 실시간 UI 하이라이트 표시
- 장애물 감지 시 자동 정지 및 제어 분기 시스템
- 아동 친화적 경고 모달 팝업 및 긴급 정지 기능
- 사용자 테스트 및 피드백 반영 (초등 3학년 역할 시뮬레이션 마일스톤 완료)
- 웹-로봇 간 데이터 유실 방지를 위한 이중 잠금(Lock/Drop) 방어 체계

### Should (우선 구현 요구사항)

- 반복 블록(Repeat) 및 조건 블록(If)의 중첩 트리 구조 구현
- 실행 차례가 온 순간 센서 상태를 판별하는 런타임 실시간 해석 아키텍처
- 라이다 전/후방 90° 광폭 부채꼴 스캔을 이용한 사각지대 차단 패치

### Could (추가 구현 요구사항)

- 멜로디 부저 연동 및 시청각 동기화(A/V Sync) 최적화
- 대용량 JSON 전송 시 웹 브라우저 메모리 누수 방지(GC) 최적화

## 3. 시스템 아키텍처

프로젝트는 크게 다음 세 부분으로 구성된다.

```text
① Web Application (UI 전체 잠금 및 가비지 컬렉션 최적화 탑재)
   - 중첩 블록 편집기 (serializeProgram 알고리즘)
   - 실행 / 긴급 정지 제어 레이아웃
   - 아동 친화적 상황 안내 모달 및 실시간 연산자 매핑 하이라이트
        │
        ▼ /program (중첩 JSON Tree)
② Interpreter Node (실시간 런타임 스택 제어 엔진)
   바퀴를 굴리며 진행하다가 블록 차례가 온 그 순간 센서값을 판별하여 해석
   - 기본 주행 제어 (전진, 후진, 좌/우회전, 대기)
   - 실시간 제어 흐름 (Repeat 루프 풀기, If 조건문 런타임 분기 주입)
   - 주행 중 실시간 도중하차 및 제어권 이양 (Runtime Skip)
        │
        ├──────────────────────────────▶ /cmd_vel (바퀴 모터 제어)
        ├──────────────────────────────▶ /buzzer (하드웨어 멜로디)
        ▼ /run_state (실시간 상태 역전파)       ▲
                                               │ /obstacle_dist/front, rear
                                               │ (90도 부채꼴 스캔망 데이터)
                                               │
                                     ③ TurtleBot 및 Lidar 센서
                                       - TurtleBot3 하드웨어 사양 검증
                                       - 라이다(LDS-03) 사각지대 제로 패치
```

당초 계획에서는 웹 앱을 서빙하는 정적 HTTP 서버까지 ROS 2 통합 런치 파일(`bringup.launch.py`) 안에서 함께 띄우는 것을 목표로 했으나, 실제 구현 단계에서는 브라우저 보안 규격 충돌을 방지하기 위해 ROS 2 노드만 launch로 묶고 웹 서버(`python3 -m http.server 8000`)는 별도 프로세스로 분리해 실행하는 이중 방어 터미널 구동 방식으로 최종 확정되었다.

## 7. 예상 결과

- **중첩 프로그래밍 플랫폼 완성:** 단순 나열식이 아닌 실제 교육용 스크래치 수준의 다중 제어 논리(반복문 내부의 조건문 등)를 로봇 하드웨어가 런타임에 실시간 연산하는 교육 환경 구축
- **시청각 동기화 체감:** 로봇 단에서 가동되는 연산자 토픽을 웹 브라우저가 다이렉트 렌더링하여 0.1초 미만의 하이라이트 동기화 완수
- **철벽 안전 방어 메커니즘 확보:** 광폭 부채꼴 스캔 및 실시간 도중하차 알고리즘 결합으로 아이들이 어떤 형태로 블록을 잘못 조립하더라도 물리적 사각지대 충돌 없이 안전하게 우회 작동하는 하이브리드 로봇 제어 기술 습득

## 9. 기대 효과

- 초등학생 사용자가 직관적인 드래그 앤 드롭을 통해 중첩 반복/조건문과 같은 고차원 컴퓨터 프로그래밍적 사고(Computational Thinking)를 실제 터틀봇 하드웨어를 통해 물리적으로 체감할 수 있다.
- 라이다 부채꼴 영역 최단거리 연산 및 런타임 제어권 분기 이양 기법을 직접 코딩해봄으로써, 임베디드 로봇 제어 환경에서 실시간 안전장치와 제어 흐름 설계에 대한 깊이 있는 역량을 축적한다.
- 웹 프론트엔드부터 ROS 2 미들웨어, 물리 하드웨어 디바이스 제어에 이르는 E2E 이종 플랫폼 간의 실시간 데이터 QoS 튜닝 및 형상 관리 협업 전반을 완벽하게 체득한다.
