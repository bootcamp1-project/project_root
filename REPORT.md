# 1. 프로젝트 개요

## 1.1 개발 배경

최근 소프트웨어 교육에서는 블록 코딩을 활용한 프로그래밍 학습이 널리 활용되고 있다. 그러나 대부분의 교육용 블록 코딩 시스템은 화면 속 가상 객체를 움직이는 수준에 머무르며, 작성한 프로그램이 실제 하드웨어에서 어떻게 동작하는지 경험하기 어렵다.

또한 기존 블록 코딩 시스템은 반복문과 조건문을 실행 전에 모두 펼쳐(Loop Unrolling) 실행하는 방식이 일반적이므로, 프로그램 실행 중 발생하는 센서 변화에 유연하게 대응하기 어렵다는 한계가 있다.

본 프로젝트는 이러한 문제를 해결하기 위해 **ROS 2 기반 TurtleBot3와 웹 애플리케이션을 연동한 블록 코딩 교육 플랫폼**을 개발하였다.

사용자가 웹에서 작성한 프로그램을 ROS 2로 전송하면, TurtleBot3가 이를 **실시간(Runtime)** 으로 해석하여 실행하도록 구현하였다.

---

## 1.2 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

- 초등학생도 쉽게 사용할 수 있는 블록 코딩 환경 제공
- 웹과 ROS 2를 연동한 실시간 로봇 제어 시스템 구현
- 반복문과 조건문의 중첩 구조 지원
- Runtime 기반 프로그램 해석 엔진 구현
- LiDAR를 이용한 안전한 자율 주행 보조 기능 구현
- 실제 로봇과 웹 화면을 실시간으로 동기화하는 교육 플랫폼 구현

---

# 2. 프로젝트 목표

프로젝트의 세부 목표는 다음과 같다.

### ① 블록 기반 프로그래밍 환경 구축

Scratch 방식과 유사한 Drag & Drop 인터페이스를 제공하여 사용자가 별도의 프로그래밍 문법을 학습하지 않고도 로봇을 제어할 수 있도록 한다.

---

### ② Runtime Tree Interpreter 구현

반복문과 조건문을 미리 평탄화하지 않고 프로그램 실행 시점에 직접 해석하는 Runtime Interpreter를 구현한다.

이를 통해 중첩 구조를 그대로 유지하면서 프로그램을 실행할 수 있도록 한다.

---

### ③ 실시간 장애물 대응

LiDAR 센서를 이용하여 전방 및 후방 장애물을 감지하고, 프로그램 실행 중에도 현재 이동 명령을 중단한 후 다음 조건문으로 자연스럽게 실행 흐름을 변경하도록 한다.

---

### ④ 웹과 로봇의 상태 동기화

현재 실행 중인 블록을 웹 화면에 실시간으로 표시하여 사용자가 로봇의 실행 상태를 직관적으로 확인할 수 있도록 한다.

---

### ⑤ 교육용 플랫폼 구현

단순히 로봇을 움직이는 것이 아니라 반복문, 조건문, 순차 실행과 같은 프로그래밍 개념을 실제 하드웨어를 통해 학습할 수 있는 교육 환경을 구축하는 것을 목표로 한다.

# 3. 시스템 아키텍처

## 3.1 전체 시스템 구성

프로젝트는 크게 다음과 같은 네 개의 계층으로 구성된다.

- Web Application
- rosbridge WebSocket Server
- ROS 2 Runtime Interpreter
- TurtleBot3 Hardware

웹 애플리케이션은 사용자가 작성한 블록 프로그램을 JSON 형태로 생성하고, `rosbridge_websocket`을 통해 ROS 2 토픽으로 전달한다.

Interpreter Node는 전달받은 프로그램을 Runtime에 해석하여 TurtleBot3를 제어하며, 실행 상태를 다시 웹으로 전달한다.

LiDAR 노드는 장애물 정보를 지속적으로 계산하여 Interpreter Node에 전달하며, Interpreter는 이를 기반으로 실행 흐름을 제어한다.

---

## 3.2 시스템 데이터 흐름

```text
              Web Application
        (Drag & Drop Block Editor)
                    │
                    │
        rosbridge_websocket
                    │
        ┌───────────┼────────────┐
        │           │            │
   /program     /run_stop   /run_state
        │           │            ▲
        ▼           ▼            │
          Interpreter Node
    (Runtime Tree Interpreter)
                    │
        ┌───────────┼────────────┐
        │           │            │
        ▼           ▼            ▼
    /cmd_vel    /buzzer     장애물 정보 구독
        │           │
        ▼           ▼
 turtlebot3_node  buzzer_node
        ▲
        │
 ultrasonic_node
        ▲
      /scan
        ▲
   TurtleBot3 LiDAR
```

---

## 3.3 시스템 동작 과정

1. 사용자가 웹에서 블록 프로그램을 작성한다.

2. 프로그램은 Nested JSON Tree 형태로 직렬화된다.

3. JSON 데이터는 rosbridge WebSocket을 통해 `/program` 토픽으로 전송된다.

4. `interpreter_node`는 프로그램을 Runtime에 해석하면서 순차적으로 블록을 실행한다.

5. 실행 중 LiDAR 데이터가 지속적으로 입력된다.

6. 장애물이 감지되면 현재 이동 블록을 중단(Skip)하고 다음 실행 흐름으로 전환한다.

7. 현재 실행 중인 Operator(op)는 `/run_state` 토픽을 통해 웹으로 전달된다.

8. 웹에서는 해당 Operator와 블록을 매핑하여 현재 실행 중인 블록을 실시간으로 하이라이트한다.

---

## 3.4 시스템 특징

본 프로젝트의 시스템은 다음과 같은 특징을 가진다.

- Runtime 기반 프로그램 해석
- 반복문 및 조건문의 중첩 구조 지원
- ROS 2와 웹의 실시간 통신
- LiDAR 기반 장애물 감지

- 실행 상태 실시간 시각화
- Web UI Lock 및 Program Drop을 통한 안정성 확보

# 6. 주요 기능

## 6.1 블록 기반 프로그래밍

사용자는 웹 브라우저에서 블록을 Drag & Drop 방식으로 배치하여 프로그램을 작성할 수 있다.

지원하는 블록은 다음과 같다.

- 전진
- 후진
- 좌회전
- 우회전
- 대기
- 부저
- 반복문(Repeat)
- 조건문(If)

반복문과 조건문은 서로 중첩하여 사용할 수 있으며, 프로그램은 Nested JSON Tree 형태로 생성된다.

---

## 6.2 Runtime Tree Interpreter

기존의 블록 코딩 시스템은 프로그램을 실행하기 전에 모든 반복문을 평탄화(Unrolling)하여 실행하는 방식이 일반적이다.

본 프로젝트에서는 이러한 방식을 사용하지 않고 Runtime Tree Interpreter를 구현하였다.

Interpreter는 실행 중 현재 블록을 해석하며 Stack 기반으로 다음 실행할 블록을 관리한다.

이를 통해 반복문과 조건문의 중첩 구조를 유지하면서 프로그램을 실행할 수 있다.

주요 특징은 다음과 같다.

- Nested JSON Tree 직접 해석
- Stack 기반 실행 구조
- Runtime 조건 판단
- 반복문 중첩 지원
- 조건문 중첩 지원

---

## 6.3 Runtime Skip & Branch

주행 중 장애물이 감지되면 프로그램 전체를 종료하지 않는다.

현재 실행 중인 이동 블록만 중단하고 다음 실행 가능한 블록으로 제어 흐름을 변경하도록 구현하였다.

```text
앞으로 이동
      │
      ▼
장애물 감지
      │
      ▼
현재 이동 명령 중단
      │
      ▼
다음 조건문 실행
```

이를 통해 프로그램의 흐름을 유지하면서도 장애물에 대응할 수 있도록 하였다.

---

## 6.4 LiDAR 기반 장애물 감지

ultrasonic_node는 TurtleBot3의 LiDAR 데이터를 이용하여 전방과 후방의 장애물 거리를 계산한다.

기존의 좁은 감지 영역 대신 전방과 후방 각각 약 90° 범위를 사용하여 장애물 감지 범위를 확장하였다.

계산된 최소 거리는

- `/obstacle_dist/front`
- `/obstacle_dist/rear`

토픽으로 Interpreter Node에 전달된다.

---

## 6.5 실행 상태 실시간 동기화

Interpreter는 현재 실행 중인 Operator(op)를 `/run_state` 토픽으로 Publish한다.

웹에서는 해당 Operator를 수신하여 실행 중인 블록을 실시간으로 하이라이트한다.

이를 통해 사용자는 현재 로봇이 어떤 블록을 수행하고 있는지 직관적으로 확인할 수 있다.

---

## 6.6 실행 중 프로그램 보호

프로그램 실행 중 사용자가 블록을 수정하면 프로그램 상태가 변경되어 예상하지 못한 동작이 발생할 수 있다.

이를 방지하기 위해 다음 기능을 구현하였다.

- 실행 중 Web UI 편집 비활성화
- 실행 중 새로운 프로그램 수신 제한
- 긴급 정지 기능 제공

# 9. 개발 과정

## Day 1 - 요구사항 분석 및 시스템 설계

프로젝트 초기 단계에서는 요구사항을 Must, Should, Could 항목으로 구분하여 우선순위를 정의하였다.

또한 웹 애플리케이션과 ROS 2 간의 데이터 흐름을 설계하고, 블록 프로그램을 Nested JSON Tree 형태로 표현하는 방식을 결정하였다.

주요 수행 내용은 다음과 같다.

# 11. 프로젝트 기여

프로젝트는 웹 애플리케이션, ROS 2, 하드웨어 제어가 동시에 동작하는 시스템으로 구성되어 있어 기능별 개발뿐만 아니라 지속적인 통합 작업이 필요하였다.

특히 Runtime Tree Interpreter를 중심으로 웹과 ROS 2, TurtleBot3가 하나의 실행 흐름으로 연결될 수 있도록 설계하였으며, 팀원 간 기능 통합과 반복적인 테스트를 통해 전체 시스템의 안정성을 확보하였다.

또한 프로젝트 전 과정에서 요구사항 분석, 시스템 설계, 구현, 통합 테스트, 문서화 과정을 반복적으로 수행하며 최종적으로 하나의 통합 플랫폼을 완성하였다.

# 12. 트러블슈팅

프로젝트를 진행하면서 웹 애플리케이션, ROS 2, TurtleBot3 간의 통합 과정에서 다양한 문제를 경험하였다. 각 문제의 원인을 분석하고 개선 과정을 거쳐 시스템의 안정성을 높였다.

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 웹과 ROS 2 통신 불안정 | WebSocket 연결 설정 문제 | rosbridge_server 기반 통신 구조를 적용하고 연결 절차를 정리 |
| 회전 각도 오차 | 실제 주행 환경과 배터리 상태에 따른 오차 | 반복 실험을 통해 회전 시간을 보정하고 상수를 조정 |
| 실행 중 하이라이트가 밀리는 문제 | 실행 순서와 화면 인덱스가 일치하지 않음 | Operator(op) 기반으로 실행 상태를 관리하여 웹과 동기화 |
| 실행 중 프로그램 수정으로 인한 오류 | 실행 중 새로운 프로그램이 동시에 전송됨 | 실행 중 UI를 잠그고 새로운 프로그램 입력을 제한 |
| 블록 전환 직후 센서 오검출 | 블록 변경 직후 센서 데이터가 즉시 반영됨 | 초기 Latch 구간을 적용하여 일시적인 센서 노이즈를 무시 |
| 장애물 감지 시 프로그램 전체 종료 | 이동 명령과 프로그램 종료 로직이 함께 처리됨 | 현재 이동 블록만 중단(Skip)하고 다음 실행 흐름으로 전환 |
| 대각선 방향 장애물 감지 부족 | 좁은 LiDAR 감지 범위 | 전·후방 감지 영역을 확장하여 장애물 감지 성능 개선 |

# 13. 프로젝트 성과

프로젝트를 통해 다음과 같은 결과를 얻을 수 있었다.

### Runtime Tree Interpreter 구현

반복문과 조건문을 실행 전에 평탄화하지 않고 Runtime에 직접 해석하는 구조를 구현하였다.

이를 통해 중첩된 블록 구조를 그대로 유지하면서 프로그램을 실행할 수 있었다.

---

### ROS 2 기반 실시간 블록 실행

웹에서 작성한 블록 프로그램을 ROS 2 Topic으로 전달하고 TurtleBot3가 이를 실시간으로 수행하는 환경을 구축하였다.

---

### Web–ROS 2 실시간 동기화

현재 실행 중인 Operator를 웹 화면과 실시간으로 동기화하여 사용자가 프로그램의 실행 상태를 직관적으로 확인할 수 있도록 하였다.

---

### 안전한 실행 환경 구축

LiDAR 기반 장애물 감지와 Runtime Skip & Branch 기능을 적용하여 장애물 발생 시 프로그램을 종료하지 않고 실행 흐름을 유지하도록 구현하였다.

또한 Web UI Lock과 Program Drop 기능을 적용하여 실행 중 발생할 수 있는 충돌을 방지하였다.

---

### 통합 시스템 구현

웹 애플리케이션, ROS 2, TurtleBot3를 하나의 시스템으로 통합하여 실제 하드웨어에서 동작하는 블록 코딩 교육 플랫폼을 구현하였다.

---

# 16. 결론

본 프로젝트에서는 웹 기반 블록 코딩 환경과 ROS 2 기반 TurtleBot3를 연동하여 실제 하드웨어를 제어할 수 있는 교육 플랫폼을 구현하였다.
